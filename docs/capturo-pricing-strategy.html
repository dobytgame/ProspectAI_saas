Como acelerar a receita
O maior alavancador é a conversão Grátis → Pro. Com 400 usuários grátis, converter apenas 33% para Pro já gera R$ 26k MRR. Estratégias: (1) limitar leads do Grátis a 100 por mês para criar urgência, (2) mostrar "você usou 87 dos 100 leads" na barra de progresso, (3) oferecer 7 dias do Pro grátis quando o usuário esgotar o plano Grátis.
Estratégia comercial
9 decisões que maximizam receita
01
Plano anual com desconto agressivo
Oferecer 2 meses grátis no anual (equivale a 17% de desconto). Melhora o LTV, reduz churn e antecipa caixa. Meta: 40% dos assinantes pagando anual.
02
Freemium como máquina de leads
O plano Grátis não é custo — é aquisição de cliente. Com 100 leads/mês, o usuário pode ter 1 fechamento real. Isso paga 1 ano do Pro. Fazer ele calcular isso na tela de upgrade.
03
Âncora Pro — evitar opção muito barata
O Starter existe para capturar quem não quer Pro ainda, mas o posicionamento do Pro como "mais popular" direciona a maioria para ele. Não dar muito no Starter.
04
Upsell automático por uso
Quando o usuário atinge 80% do limite de leads, exibir notificação: "Você está gerando ótimos resultados. Faça upgrade para continuar sem interrupções." Momento de maior intenção.
05
Add-on de leads extra
Vender pacotes extras: 200 leads = R$ 47, 500 leads = R$ 97. Gera receita variável e teste de valor antes de upgrade. Usuário Pro que compra extra tende a virar Scale.
06
Garantia de 7 dias — reduz risco percebido
"Se não fechar nenhum lead em 7 dias, devolvemos 100%." Reduz a objeção de preço. Em SaaS com produto bom, taxa real de reembolso é menor que 3%.
07
Indicação com comissão
Programa de afiliados: 20% recorrente por 6 meses para quem indicar. Um afiliado ativo com 10 referências gera R$ 394/mês passivo — ele vira vendedor voluntário.
08
Cupom para grupos e comunidades
Código "AGENCIAS20" (20% off primeiro mês) para grupos de WhatsApp de agências e closers. Custo zero de aquisição, alta conversão em comunidades nichadas.
09
Trial Pro para usuários Grátis ativos
Usuário que usa o Grátis por 2+ semanas recebe e-mail: "Você está mandando bem. Aqui estão 7 dias do Pro, de graça." Taxa de conversão de trial ativo é 4× maior que cold upgrade.
Implementação técnica
Como colocar isso no produto
Migration SQL — campo de plano
alter table public.businesses
  add column if not exists plan text
    default 'free',
  add column if not exists plan_leads_used int
    default 0,
  add column if not exists plan_expires_at
    timestamptz,
  add column if not exists stripe_customer_id
    text;

-- Limites por plano
create or replace function
  get_plan_limits(p_plan text)
returns json as $$
  select case p_plan
    when 'free'    then
      json_build_object('leads',100,
        'campaigns',1,'users',1)
    when 'starter' then
      json_build_object('leads',300,
        'campaigns',3,'users',1)
    when 'pro'     then
      json_build_object('leads',1000,
        'campaigns',-1,'users',1)
    when 'scale'   then
      json_build_object('leads',5000,
        'campaigns',-1,'users',3)
    else
      json_build_object('leads',100,
        'campaigns',1,'users',1)
  end
$$ language sql;
Checagem de limite em lead-actions.ts
// Antes de buscar leads, verificar limite
const PLAN_LIMITS: Record<string, number> = {
  free:    100,
  starter: 300,
  pro:     1000,
  scale:   5000,
};

const { data: biz } = await supabase
  .from("businesses")
  .select("id, icp, plan, plan_leads_used, agents(config)")
  .eq("user_id", user.id)
  .single();

const limit = PLAN_LIMITS[biz.plan || 'free'];
const used = biz.plan_leads_used || 0;

if (used >= limit) {
  throw new Error(
    `LIMIT_REACHED:${limit}:${biz.plan}`
  );
}
