IA — Prompts · Item 2
Análise do Negócio — trainAgent()
O prompt atual é básico demais: pede ICP, tom e nicho. Falta ensinar a IA a pensar como um vendedor expert que conhece profundamente o negócio. O novo prompt extrai muito mais contexto e cria um verdadeiro "DNA de vendas" para o agente.
Problemas no prompt atual
FIX
Sem contexto de vendas — IA não sabe o que fazer com o ICP
O prompt atual pede apenas "target_audience" e "pain_points" sem dizer para a IA como usar essas informações na prospecção. O agente resultante é genérico.
FIX
Falta: gatilhos de compra, objeções comuns, proposta de valor única
Sem saber quando o cliente compra e por que não compra, o agente não consegue gerar mensagens persuasivas ou responder objeções.
NEW
Adicionar: persona do vendedor + estilo de abordagem por segmento
O agente precisa ter uma identidade de vendas clara. Quem ele é, como fala, o que nunca diz.

O scoring atual não usa os novos campos do trainAgent (buying_triggers, red_flags). Atualizar para aproveitar todo o DNA de vendas.
src/lib/ai/claude.ts — scoreLead()
TypeScript 
/**
 * SCORING DE LEADS — Versão 2.0
 * Usa o DNA completo do agente: icp + scoring_criteria + buying_triggers.
 */
export async function scoreLead(leadData: any, agentConfig: any) {
  const icp = agentConfig?.icp || agentConfig;
  const criteria = agentConfig?.scoring_criteria || {};
  const triggers = icp?.buying_triggers || [];

  const prompt = `Você é um especialista em qualificação de leads B2B.
Avalie o fit deste lead com o ICP do negócio e dê um score de 0 a 100.

ICP DO NEGÓCIO:
${JSON.stringify(icp, null, 2)}

CRITÉRIOS DE SCORING:
Must have (elimina se ausente): ${JSON.stringify(criteria.must_have || [])}
Nice to have (aumenta score): ${JSON.stringify(criteria.nice_to_have || [])}
Red flags (diminui score): ${JSON.stringify(criteria.red_flags || [])}
Gatilhos de compra a detectar: ${JSON.stringify(triggers)}

DADOS DO LEAD:
- Nome: ${leadData.name}
- Categorias Google: ${JSON.stringify(leadData.metadata?.types || [])}
- Avaliação: ${leadData.metadata?.rating || 'N/A'}/5 (${leadData.metadata?.user_ratings_total || 0} avaliações)
- Endereço: ${leadData.address}
- Tem website: ${leadData.website ? 'Sim — ' + leadData.website : 'Não'}
- Tem telefone: ${leadData.phone ? 'Sim' : 'Não'}

ESCALA:
- 86-100: Lead perfeito — todos os must_have, nenhum red_flag
- 71-85: Bom fit — maioria dos critérios atendidos
- 51-70: Potencial — fit parcial, vale tentar
- 31-50: Baixo fit — alguns critérios incompatíveis
- 0-30: Desqualificado — red_flag eliminatório presente

Retorne APENAS JSON (sem markdown):
{
  "score": number,
  "tier": "A+|A|B|C|D",
  "reasoning": "explicação objetiva em 1 frase",
  "fit_reasons": ["ponto positivo 1", "ponto 2"],
  "concerns": ["preocupação 1"],
  "recommended_angle": "qual dos ângulos de abordagem usar com este lead",
  "priority": "high|medium|low"
}`;

  const response = await anthropic.messages.create({
    model: "claude-sonnet-4-5",
    max_tokens: 500,
    messages: [{ role: "user", content: prompt }],
  });

  const content = response.content[0];
  if (content.type === "text") {
    try { return parseClaudeJSON(content.text); }
    catch { return { score: 50, tier: "C", reasoning: "Falha na análise.", fit_reasons: [], concerns: [], priority: "medium" }; }
  }
  return { score: 0, tier: "D", reasoning: "Erro de comunicação.", fit_reasons: [], concerns: [], priority: "low" };
}

Geração de Mensagem — generateOutreachMessage()
O prompt atual tem boas regras mas não transforma o agente em um vendedor. Não usa os ângulos de abordagem, a persona de vendas nem o recommended_angle do scoring. A mensagem resultante é boa mas não é extraordinária.

/**
 * GERAÇÃO DE MENSAGEM — Versão 2.0
 * Usa persona de vendas, ângulo recomendado pelo scoring e tom calibrado.
 * Cada mensagem é única, específica e soa como escrita por um humano expert.
 */
export async function generateOutreachMessage(
  lead: any, business: any, campaign: any, agentConfig?: any
) {
  const isWhatsApp = campaign.channel === 'whatsapp';
  const persona = agentConfig?.sales_persona || {};
  const angles = agentConfig?.approach_angles || [];
  const recommendedAngle = lead.metadata?.recommended_angle || angles[0] || '';
  const objections = agentConfig?.objection_handling || {};
  const neverSay = persona.never_say || [];

  const systemPrompt = `Você é ${persona.name || business.name}, ${persona.role || 'Consultor de Vendas'}.
Sua personalidade: ${persona.personality || business.tone}.
Você representa: ${business.name} — ${business.segment}.

REGRAS DE OURO da sua comunicação:
${neverSay.map((s: string, i: number) => `${i+1}. NUNCA diga: "${s}"`).join('\n')}
- NUNCA mencione automação, robô ou IA
- NUNCA use linguagem corporativa genérica
- SEMPRE soe como uma pessoa real que pesquisou o lead
- SEMPRE foque no resultado para o lead, não nas features do produto`;

  const userPrompt = `Escreva UMA mensagem de primeiro contato para o lead abaixo.

CONTEXTO DA CAMPANHA:
Canal: ${campaign.channel.toUpperCase()}
Objetivo: ${campaign.name} — ${campaign.description}

DADOS DO LEAD (use para personalizar):
- Empresa: ${lead.name}
- Endereço: ${lead.address}
- Avaliação Google: ${lead.metadata?.rating || 'N/A'} estrelas (${lead.metadata?.user_ratings_total || 0} avaliações)
- Score de qualificação: ${lead.score}/100 (${lead.metadata?.tier || 'B'})
- Por que é um bom fit: ${lead.metadata?.reasoning || ''}
- Pontos fortes do lead: ${JSON.stringify(lead.metadata?.fit_reasons || [])}
${lead.phone ? `- Telefone: ${lead.phone}` : ''}
${lead.website ? `- Website: ${lead.website}` : ''}

ÂNGULO DE ABORDAGEM RECOMENDADO:
${recommendedAngle || 'Use o ângulo mais relevante para o perfil do lead'}

PROPOSTA DE VALOR DO PRODUTO:
${business.icp?.solution_value || ''}

${isWhatsApp ? `FORMATO WHATSAPP:
- Máximo 3 parágrafos CURTOS (cada um com no máximo 2 linhas)
- Tom conversacional, direto, sem formalidades excessivas
- Sem formatação markdown (sem **, sem -, sem listas)
- Terminar com UMA pergunta aberta simples
- Assinatura: ${persona.name || business.name}` : `FORMATO EMAIL:
- Primeira linha: ASSUNTO: [assunto que gera curiosidade, máximo 50 chars]
- 3-4 parágrafos bem estruturados
- Abertura que não começa com "Olá" ou "Oi"
- Parágrafo de valor claro com dado ou resultado específico
- CTA claro e fácil de responder
- Assinatura: ${persona.name || business.name} | ${business.name}`}

IMPORTANTE: A primeira frase DEVE mencionar algo específico sobre o ${lead.name} ou sua localização.
Escreva APENAS a mensagem. Sem explicações, sem comentários, sem notas.`;

  const response = await anthropic.messages.create({
    model: "claude-sonnet-4-5",
    max_tokens: 800,
    system: systemPrompt,
    messages: [{ role: "user", content: userPrompt }],
  });

  const content = response.content[0];
  if (content.type === "text") return content.text;
  throw new Error("Falha ao gerar mensagem.");
}

import { NextResponse } from 'next/server'
import Anthropic from "@anthropic-ai/sdk";
import { createClient } from "@/utils/supabase/server";

const anthropic = new Anthropic({ apiKey: process.env.CLAUDE_API_KEY });

export async function POST(req: Request) {
  try {
    const { message, leadId, history = [] } = await req.json()
    const supabase = await createClient()

    // Busca lead + business + agentConfig completo
    const { data: lead } = await supabase
      .from("leads")
      .select("*, businesses(*, agents(config))")
      .eq("id", leadId)
      .single()

    if (!lead) return NextResponse.json({ reply: "Lead não encontrado." })

    const business = lead.businesses
    const agentConfig = business.agents?.[0]?.config || {}
    const persona = agentConfig.sales_persona || {}
    const angles = agentConfig.approach_angles || []
    const objections = agentConfig.objection_handling || {}
    const icp = agentConfig.icp || business.icp || {}

    const systemPrompt = `Você é ${persona.name || 'Agente ' + business.name}, ${persona.role || 'Assessor Comercial'}.
Você representa ${business.name} — especialistas em ${business.segment}.
Personalidade: ${persona.personality || business.tone || 'Consultivo e direto'}.

SEU OBJETIVO:
Ajudar o usuário a prospectar e fechar negócio com: ${lead.name}

CONTEXTO DO LEAD:
- Empresa: ${lead.name}
- Endereço: ${lead.address}
- Score de qualificação: ${lead.score}/100 (Tier ${lead.metadata?.tier || 'B'})
- Análise da IA: ${lead.metadata?.reasoning || ''}
- Pontos positivos: ${JSON.stringify(lead.metadata?.fit_reasons || [])}
- Pontos de atenção: ${JSON.stringify(lead.metadata?.concerns || [])}
${lead.phone ? `- Telefone: ${lead.phone}` : ''}
${lead.website ? `- Website: ${lead.website}` : ''}

CONTEXTO DO NEGÓCIO:
- ICP: ${JSON.stringify(icp)}
- Ângulos de abordagem disponíveis: ${JSON.stringify(angles)}
- Manejo de objeções treinado: ${JSON.stringify(objections)}

COMO VOCÊ DEVE AGIR:
1. Sugira scripts de mensagem prontos para usar quando pedido
2. Analise respostas do lead e oriente o próximo passo
3. Antecipe objeções usando o manejo treinado
4. Recomende o canal certo (WhatsApp/Email/visita) baseado no score
5. Seja CONCISO — máximo 150 palavras por resposta
6. Use bullets quando listar itens (máximo 4 itens)
7. NUNCA revele que é um agente de IA ou automação`;

    // Histórico de conversa (enviado pelo client)
    const messages = [
      ...history.map((h: any) => ({ role: h.role, content: h.content })),
      { role: "user" as const, content: message }
    ];

    const response = await anthropic.messages.create({
      model: "claude-sonnet-4-5",
      max_tokens: 600,
      system: systemPrompt,
      messages: messages,
    });

    const content = response.content[0];
    const reply = content.type === "text" ? content.text : "Erro interno.";
    return NextResponse.json({ reply })

  } catch (error) {
    console.error("Chat error:", error)
    return NextResponse.json({ reply: "Erro ao processar." }, { status: 500 })
  }
}

/** Salva uma nota no metadata do lead */
export async function saveLeadNote(leadId: string, note: string) {
  const supabase = await createClient();
  const { data: lead } = await supabase
    .from("leads").select("metadata").eq("id", leadId).single();

  const history = lead?.metadata?.contact_history || [];
  history.push({ type: 'note', content: note, at: new Date().toISOString() });

  await supabase.from("leads")
    .update({ metadata: { ...lead?.metadata, notes: note, contact_history: history } })
    .eq("id", leadId);
  revalidatePath("/pipeline");
}

/** Adiciona ou remove uma tag do lead */
export async function toggleLeadTag(leadId: string, tag: string) {
  const supabase = await createClient();
  const { data: lead } = await supabase
    .from("leads").select("metadata").eq("id", leadId).single();

  const tags: string[] = lead?.metadata?.tags || [];
  const newTags = tags.includes(tag)
    ? tags.filter(t => t !== tag)
    : [...tags, tag];

  await supabase.from("leads")
    .update({ metadata: { ...lead?.metadata, tags: newTags } })
    .eq("id", leadId);
  revalidatePath("/pipeline");
}

/** Registra um contato manual (chamada, reunião, etc.) */
export async function logContact(leadId: string, type: string, summary: string) {
  const supabase = await createClient();
  const { data: lead } = await supabase
    .from("leads").select("metadata").eq("id", leadId).single();

  const history = lead?.metadata?.contact_history || [];
  history.push({ type, summary, at: new Date().toISOString() });

  await supabase.from("leads")
    .update({ metadata: { ...lead?.metadata, contact_history: history } })
    .eq("id", leadId);
  revalidatePath("/pipeline");
}

function RecentActivity({ leads }: { leads: Lead[] }) {

  };

      <span className="text-xs font-bold uppercase tracking-wider block mb-4"
        style={{ color: 'var(--foreground-muted)' }}>Atividade Recente</span>

          return (
            <div key={l.id} className="flex items-center gap-3">
              <div className="w-2 h-2 rounded-full shrink-0"
                style={{ background: cfg.color, boxShadow: `0 0 6px ${cfg.color}60` }} />
              <div className="flex-1 min-w-0">
                <span className="text-xs font-medium block truncate"
                  style={{ color: 'var(--foreground)' }}>{l.name}</span>

              </div>

            style={{ color: 'var(--foreground-dim)' }}>Nenhuma atividade ainda</p>

alter table public.leads
  add column if not exists updated_at timestamp with time zone
  default timezone('utc'::text, now());

create or replace function update_updated_at()
returns trigger as $$
begin
  NEW.updated_at = timezone('utc'::text, now());
  return NEW;
end;

create trigger leads_updated_at_trigger
  before update on public.leads

  select json_build_object(

        from public.leads
        where business_id = p_business_id
        order by score desc
        limit 5
      ) sub
    )
  )

  where business_id = p_business_id
